# 《Moonlit Draw》需求文档

## 1. Overview

《Moonlit Draw》是一款月夜武士单点反应决斗游戏：玩家扮演画面左侧持刀者，等待月光闪出 `STRIKE!` 后抢先点击，连续击败越来越快的对手。

## 2. Visual Design

- 全屏 3D 月夜道场，冷蓝月光、鸟居剪影、灯笼与低多边形持刀者构成主要视觉。
- 首次进入先用约 1.63 秒镜头从左侧玩家近景平滑拉到双人全景；玩家头顶显示 `YOU` 标签和向下箭头，开局后保留 1.1 秒再淡出。
- 标题位于顶部安全区，等待提示位于场地中心，点击提示位于底部 16% 安全区；高度小于 650 px 时标题区上收至 24 px。
- 完整颜色、排版、动效和禁用模式见 `doc/visual.md`。
- 素材：`public/poster.png` 用于封面；角色、道场、刀和粒子均由 Three.js 几何体实时渲染。

## 3. Game Mechanics

- 玩家与对手分别位于 `x=-1.05` 与 `x=1.05`，角色缩放为 `0.78`，相机 FOV 为 `46°`。
- 每局等待时间随机为 `1.05–3.25` 秒；开局前 `0.42` 秒为防误触保护期。
- 首名对手反应约 `0.56±0.026` 秒，每胜一局缩短 `0.033` 秒，最低约 `0.16±0.026` 秒。
- `requestAnimationFrame` 驱动相机、待机呼吸、尘埃、刀光、慢动作和 120 个碎片粒子。
- 首次进入为 `ATTRACT` 状态，不启动失败倒计时；第一次点击只开始决斗，不会判破势。

## 4. Controls

- Pointer：任意非榜单区域 `pointerdown`。首次点击开始；等待信号时过早点击判犯规；出现 `STRIKE!` 后点击记录反应时间。
- Click：结算页的再来一次、查看排行，以及榜单行使用 `click`。
- Keyboard / Drag：不支持，也不需要悬停操作。

## 5. Win / Lose Conditions

- 玩家在对手反应时间前点击则赢得本局，胜场加 1，并进入下一名更快的对手。
- 玩家在保护期后、`STRIKE!` 前点击则 `BROKE FORM!`，立即失败。
- 玩家点击晚于对手或未在对手反应时间内点击则失败。
- 结算页展示连续胜场、最佳出刀、击败玩家的对手、再来一次和排行榜入口。

## 6. Sound Effects

- 等待：55 Hz 锯齿低频持续音，加 72→44 Hz 与 60→38 Hz 双心跳，间隔由 0.95 秒加速至 0.42 秒。
- `STRIKE!`：1700 Hz 方波、2550 Hz 正弦与 320→1300 Hz 上升锯齿音，持续 `80–180 ms`。
- 出刀：高通噪声、980→1720 Hz 三角波与 540→240 Hz 余韵，持续 `45–160 ms`。
- 胜利：523/659/784/1047 Hz 上行音阶；失败：230→80 Hz 下行音；抢跑：200→90 Hz 方波与低频噪声。
